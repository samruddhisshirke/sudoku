name: Python Tests

on:
  push:
    branches:
      - main

  pull_request:
    branches:
      - main

jobs:
  test:

    runs-on: ubuntu-latest

    steps:

    - name: Checkout Repository
      uses: actions/checkout@v4

    - name: Set up Python
      uses: actions/setup-python@v5
      with:
        python-version: '3.11'

    - name: Install Dependencies
      run: |
        python -m pip install --upgrade pip
        pip install pytest

    - name: Run Tests
      run: pytest

Exp 1: Git Initialization & Commit
1. What is Git? Git is a distributed version control system used to track code changes.
2. What does `git init` do? It creates a new local Git repository.

Exp 2: Push Local Project to GitHub 
3. What is GitHub? GitHub is a cloud platform for hosting Git repositories.
4. What does `git push` do? Uploads local commits to remote repository.

Exp 3: Git Branching & Merging
5. What is a branch in Git? A separate line of development in a repository.
6. What is merging? Combining changes from one branch into another.

Exp 4: GitHub Actions Basic Workflow 
7. What is GitHub Actions? CI/CD automation service provided by GitHub.
8. What is a workflow file? A YAML file defining automation steps.

Exp 5: GitHub Actions on Push
9. What triggers a workflow? Events like push, pull request, etc.
10. What is CI? Continuous Integration — automatic testing/building after code changes.

Exp 6: GitHub Actions with HTML/Webpage
11. What is `index.html`? Main webpage file of a website.
12. Why use YAML in GitHub Actions? To define workflow configuration.

Exp 7: Automated Testing using Pytest 
13. What is Pytest? A Python testing framework.
14. Why did pytest show “0 tests collected”? Because test file/function naming convention was incorrect.

Exp 8: Dockerfile & Docker Image
15. What is Docker? Docker is a containerization platform.
16. What is a Dockerfile? A file containing instructions to build Docker image.

Exp 9: Docker Compose
17. What is Docker Compose? Docker Compose is used to manage multiple containers together.
18. What is the purpose of `docker-compose.yml`? Defines services, ports, and container configuration.

Exp 10: Ansible Playbook
19. What is Ansible? Ansible is used for configuration management and automation.
20. What is a playbook in Ansible? A YAML file containing automation tasks.
